
## Understanding the Current State

I've read through everything. Here's what exists today:

### Stellar Engine (L0 candidate)
- A fork of Google Cloud Foundation Fabric (CFF) providing IaC for GCP landing zones with Assured Workload overlays (FedRAMP High, IL4, IL5)
- **FAST stages**: 0-Bootstrap → 1-Resource Management → 2-Networking → 3-Security
- **Modules**: ~80+ Terraform modules (VPC, IAM, GKE, BigQuery, KMS, etc.)
- **Blueprints**: ~35 compliance-mapped blueprints (FRH ✅, IL5 ✅)
- Target audience: Government agencies, regulated industries, universities

### Workload-Viz Repos (L1/L2 candidates)
Each follows the same pattern with two boxes:
1. **IT Box** (left side): 6 automated infra steps (Enable APIs → Create SA → IAM Roles → Org Policies → Create VPC → Cloud NAT) — these are provisioned by IT via SSE streaming
2. **Researcher Box** (right side): Domain-specific pipeline that researchers run

Three exist today:
- `workload-viz-nextflow-batch` — Nextflow RNAseq on Cloud Batch
- `workload-viz-medsiglip-pathology` — MedSigLIP fine-tuning on Vertex AI Workbench (A100)
- `workload-viz-toxicology` — DILI prediction via RAG + TxGemma + GROMACS MD

## Proposed 3-Tier Architecture: "Fast Science"

### L0 — `stellar-engine` (Central IT Foundation)
**Owned by:** Central IT / Cloud Platform Team  
**Purpose:** Organization-level GCP landing zone — the "ground floor" that must exist before anything else

**What changes in this repo:**
- **Opinionated README rewrite** — Replace the current wall-of-text README with a focused step-by-step deployment guide:
  1. Prerequisites (Org, billing, IAM admin)
  2. Stage 0: Bootstrap (step-by-step, no mega bash scripts)
  3. Stage 1: Resource Management
  4. Stage 2: Networking (pick FRH or IL5)

  6. "Your landing zone is ready. Now provision researcher projects via L1."

### L1 — `researcher-lab` (IT-Provisioned Researcher Projects)  
**Owned by:** Central IT, cloned per university  
**Purpose:** This is the "IT box" from your workload-viz repos, extracted into reusable Terraform modules

**Folder Structure:**
```
researcher-lab/
├── README.md                           # Step-by-step: "How IT provisions a researcher project"
├── CommonComponents/
│   ├── ApiAllowlistModule/
│   │   ├── main.tf                     # Opinionated API enablement
│   │   ├── variables.tf                # researcher_name, project_id, allowed_apis
│   │   └── outputs.tf
│   ├── ServiceAccountModule/
│   │   ├── main.tf                     # SA creation with least-privilege
│   │   ├── variables.tf
│   │   └── outputs.tf
│   ├── IAMRolesModule/
│   │   ├── main.tf                     # Role bindings per researcher persona
│   │   ├── variables.tf
│   │   └── outputs.tf
│   ├── NetworkModule/
│   │   ├── main.tf                     # VPC + subnet + Cloud NAT + firewall
│   │   ├── variables.tf
│   │   └── outputs.tf
│   ├── OrgPoliciesModule/
│   │   ├── main.tf                     # Org policy overrides (e.g., shielded VM for GPU)
│   │   ├── variables.tf
│   │   └── outputs.tf
│   └── BudgetAlertModule/
│       ├── main.tf                     # Budget alerts per researcher project
│       ├── variables.tf
│       └── outputs.tf
│
├── researchers/                         # One .tf file per researcher/lab — the "factory"
│   ├── JohnDoe-DeptOfLifeSciences-Genomics/
│   │   └── main.tf
│   │   # module "apis"  { source = "../../CommonComponents/ApiAllowlistModule" ... }
│   │   # module "sa"    { source = "../../CommonComponents/ServiceAccountModule" ... }
│   │   # module "iam"   { source = "../../CommonComponents/IAMRolesModule" ... }
│   │   # module "net"   { source = "../../CommonComponents/NetworkModule" ... }
│   │   # var: blueprint = "nextflow"  ← links to L2 catalog
│   │
│   ├── PaulUnderwood-DeptOfLifeSciences-Genomics/
│   │   └── main.tf
│   │
│   └── FlameInstitute-DeptOfCompSci-JaxResearch/
│       └── main.tf
│
└── templates/
    └── researcher-template.tf           # Copy-paste template for new researchers
```

**Key insight:** Each researcher's `.tf` file is essentially a declaration like:
```hcl
module "john_doe_project" {
  source = "../../CommonComponents/..."
  
  researcher_name  = "john-doe"
  department       = "life-sciences"
  research_area    = "genomics"
  budget_monthly   = 5000
  
  # Which APIs they need (opinionated defaults + overrides)
  api_allowlist    = ["aiplatform", "batch", "compute", "storage"]
  
  # Which L2 blueprint they'll use
  blueprint        = "nextflow"
}
```

## How They Connect

```
L0: stellar-engine (Central IT)
│   "Deploy GCP org landing zone"
│   Stages 0→1→2→3 = Org, Folders, Networking, Security
│
└──> L1: researcher-lab (Central IT per-university)
     │   "Provision researcher projects within the landing zone"
     │   CommonComponents/ = Reusable TF modules
     │   researchers/JohnDoe/ = Per-researcher project declarations
     │
     └──> L2: fast-science-blueprints (Researcher)
          │   "Run your science workload"
          │   Pick: Nextflow | MedSigLIP | Toxicology | CryoEM | ...
          │   Assumes: Project exists, APIs on, SA created, VPC ready
          └── Just run: backend + frontend + notebook
```
